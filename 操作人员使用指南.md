# 操作人员使用指南

本文档说明如何完整跑通车辆轮式分类流程：准备数据、切分数据、训练模型、识别图片。

## 一、环境准备

确认电脑已安装 Python 3，并安装以下依赖：

```bash
pip install torch torchvision pillow
```

如果系统同时存在 Python 2 和 Python 3，请使用 `python3` 和 `pip3`。不要用指向 Python 2 的 `python` 命令运行本项目。

## 二、准备数据

只需要维护 `data/raw` 目录。按车辆类别把图片放到对应文件夹：

```text
data/raw/two_wheel/     # 二轮车
data/raw/three_wheel/   # 三轮车
data/raw/four_wheel/    # 四轮车
data/raw/other/         # 其他（非车图片，用于拒识）
```

建议每类至少 200 张。图片要求：

1. 主体车辆清楚。
2. 尽量一张图只包含一辆车。
3. 删除错图、糊图、玩具图、卡通图、图标和无关图片。
4. 三类数量尽量接近。

`other` 类的作用是让模型学会"这不是我要识别的三种车"。如果不放 `other` 图片，模型对完全不是车的输入会强行三选一。`other` 里建议放：

1. 人物、动物、风景、室内、食物等明显非车图片。
2. 玩具车、卡通车、车标、图标等不希望被识别成真车的图片。
3. 车的零件特写（如单独的轮子、车灯）、火车、飞机、船只等其他交通工具。
4. 婴儿车、购物车、轮椅等带轮子但不属于三类车的图片。

不要全放风景，尽量覆盖容易和真车混淆的场景。建议数量与其他三类接近（200 张左右）。

不要手动维护 `data/train` 和 `data/val`，这两个目录由脚本生成。

## 三、生成训练集和验证集

在项目根目录执行：

```bash
python3 split_manual_data.py
```

脚本会从 `data/raw` 复制图片，生成：

```text
data/train/
data/val/
```

默认约 80% 图片进入训练集，20% 图片进入验证集。这个操作可以重复执行，每次都会重新生成 `train` 和 `val`，不会修改 `raw`。

## 四、训练模型

执行：

```bash
python3 train_real.py
```

训练完成后会生成或更新：

```text
wheelnet_model.pth
```

终端会输出每轮训练结果，例如：

```text
epoch 1/5 train_loss=1.0100 val_acc=0.7637 [二轮车=0.68 三轮车=0.43 四轮车=0.98 其他=0.92]
epoch 5/5 train_loss=0.4458 val_acc=0.8297 [二轮车=0.80 三轮车=0.64 四轮车=0.98 其他=0.88]
best_val_acc=0.8462
saved: wheelnet_model.pth

混淆矩阵（行=真实, 列=预测）:
           二轮车  三轮车  四轮车  其他
  二轮车       34     7     0     0
  三轮车        9    30     5     0
  四轮车        0     0    46     1
  其他        3     2     1    44
```

`best_val_acc` 是当前验证集准确率。每个 epoch 行末方括号里是各类别在验证集上的准确率，方便快速看出哪一类拖后腿。

训练结束打印的混淆矩阵中，行是真实类别，列是预测类别。对角线是分类正确的张数，其他位置表示错分情况。例如上面 `二轮车` 行 `三轮车` 列等于 7，表示有 7 张真实的二轮车被错判成三轮车。混淆矩阵能告诉你接下来该重点补充哪一类数据。

## 五、运行内置演示

执行：

```bash
python3 wheelnet.py
```

程序会加载 `wheelnet_model.pth`，对内置演示图片做一次识别。

## 六、识别指定图片

识别某一张图片：

```bash
python3 wheelnet.py /path/to/image.jpg
```

也可以使用参数形式：

```bash
python3 wheelnet.py --image-path /path/to/image.jpg
```

macOS / Linux 示例：

```bash
python3 wheelnet.py /var/tmp/test_car.jpg
```

Windows 示例：

```bash
py -3 wheelnet.py C:\Users\operator\Pictures\test_car.jpg
```

输出示例：

```text
wheelnet 单图识别
图片: /var/tmp/test_car.jpg
识别结果: 四轮车
置信度: 88.42%
尺寸: 640x427x3
设备: cpu
```

如果输入的不是三类车的图片（例如风景、人物、玩具车），识别结果可能是 `其他`。这表示模型认为该图不属于二/三/四轮车中的任何一种。

也可以把单张图片或整个目录的识别结果保存成 JSON：

```bash
python3 wheelnet.py /path/to/image.jpg --json-output result.json
python3 wheelnet.py /path/to/images --json-output result.json
```

如果要递归识别子目录：

```bash
python3 wheelnet.py /path/to/images --recursive --json-output result.json
```

JSON 中每张图片会记录文件路径、文件名、图片所在目录全路径、图片宽高、预测类别、类别英文码和置信度。当前模型是整图分类模型，不输出目标位置框，所以 `bbox` 字段为 `null`。

目录识别时，终端会逐张显示正在识别的文件、识别结果、置信度和完成进度。

如果提示 `未找到 wheelnet_model.pth`，请先执行训练命令：

```bash
python3 train_real.py
```

## 七、标准完整流程

每次新增或清洗图片后，按这个顺序执行：

```bash
python3 split_manual_data.py
python3 train_real.py
python3 wheelnet.py /path/to/test_image.jpg
```

其中 `/path/to/test_image.jpg` 换成要识别的实际图片路径。

## 八、常见问题

**1. 能不能直接把新图片放到 `data/train`？**

不建议。请放到 `data/raw`，然后重新运行 `split_manual_data.py`。

**2. 为什么同一张图片置信度不高？**

可能是训练数据不足、图片角度特殊、主体太小、背景干扰大，或者图片和训练集风格差异太大。

**3. 为什么重新训练结果有轻微变化？**

训练中包含随机数据增强，不同系统和依赖版本也可能造成小幅差异。数据切分已经固定，但训练结果不保证逐位完全一致。

**4. 识别结果是「其他」怎么办？**

说明模型认为这张图不属于二/三/四轮车。常见原因：

- 图片确实不是这三类车（例如风景、人、玩具）→ 这是预期行为。
- 图片是真实车辆，但被错判成其他 → 检查 `data/raw/other/` 是否混入了和真车相似的图片，必要时移走再重新训练。
- 车在图里太小、被严重遮挡、角度特殊 → 模型不确定时倾向于判到 `其他`。

**5. 模型改成 4 类以后，旧的 `wheelnet_model.pth` 还能用吗？**

不能。新代码加载旧模型时会报错"模型类别不一致"。请删除旧的 `wheelnet_model.pth`，重新跑一次 `python3 train_real.py`。
