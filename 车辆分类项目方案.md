# 车辆分类项目方案

> 2026-05-21 更新

## 一、项目目标

本项目用于识别图片中的车辆轮式类别，输出三类结果：

| 类别目录 | 中文结果 | 说明 |
|---|---|---|
| `two_wheel` | 二轮车 | 自行车、电动车、摩托车等 |
| `three_wheel` | 三轮车 | 三轮车、三轮摩托、三轮货运车等 |
| `four_wheel` | 四轮车 | 轿车、SUV、货车、客车等 |

当前项目不再依赖自动爬图。图片由操作人员手动下载、筛选、归类，程序只负责稳定切分数据、训练模型和执行识别。

## 二、当前技术路线

核心方案是 **MobileNetV2 迁移学习 + 手工整理数据集**：

1. 使用 ImageNet 预训练的 MobileNetV2 作为特征提取骨干。
2. 替换最后分类层，输出二轮、三轮、四轮 3 类。
3. 冻结 backbone，只训练最后分类头，适合无 GPU 的普通电脑。
4. 使用固定随机种子从 `data/raw` 复制生成 `data/train` 和 `data/val`。
5. 训练过程中保存验证集准确率最高的模型到 `wheelnet_model.pth`。
6. 命令行可指定任意图片路径，输出中文识别结果和置信度。

## 三、数据目录规范

人工维护的数据只放在 `data/raw`：

```text
data/
  raw/
    two_wheel/
    three_wheel/
    four_wheel/
```

`data/train` 和 `data/val` 是脚本生成的中间目录，不作为人工维护目录：

```text
data/
  train/
    two_wheel/
    three_wheel/
    four_wheel/
  val/
    two_wheel/
    three_wheel/
    four_wheel/
```

当前数据量：

```text
raw/two_wheel    208
raw/three_wheel  221
raw/four_wheel   238
```

建议每类至少 200 张起步，后续如果要继续提高准确率，优先补充容易错分的类别和场景。

## 四、训练流程

标准流程：

```bash
python3 split_manual_data.py
python3 train_real.py
python3 wheelnet.py
```

含义：

1. `split_manual_data.py` 从 `data/raw` 读取图片，按固定随机种子复制生成 `data/train` 和 `data/val`。
2. `train_real.py` 读取训练集和验证集，训练 MobileNetV2 三分类模型。
3. `wheelnet.py` 加载 `wheelnet_model.pth`，运行内置演示或识别单张图片。

最近一次固定切分后的数量：

```text

val/two_wheel       41
train/three_wheel  177
val/three_wheel     44
train/four_wheel   191

```

最近一次训练结果：

```text
best_val_acc=0.8333

```

该准确率基于当前验证集，不代表所有真实场景效果。后续应通过增加数据、清洗错图和查看错分样本继续改进。

## 五、识别方式

运行内置演示：

```bash
python3 wheelnet.py
```

识别单张图片：

```bash
python3 wheelnet.py /path/to/image.jpg
```

或者：

```bash
python3 wheelnet.py --image-path /path/to/image.jpg
```

输出示例：

```text
wheelnet 单图识别
图片: /path/to/image.jpg
识别结果: 三轮车
置信度: 92.35%
设备: cpu
```

## 六、代码文件说明

| 文件 | 作用 |
|---|---|
| `split_manual_data.py` | 从 `data/raw` 稳定生成 `data/train` 和 `data/val` |
| `train_real.py` | 训练真实数据模型，输出 `wheelnet_model.pth` |
| `wheelnet.py` | 加载模型，运行演示或识别单张图片 |
| `collect_data.py` | 早期自动采集脚本，当前流程不推荐依赖 |
| `tests/test_collect_data.py` | 采集脚本的回归测试 |

## 七、数据质量要求

训练效果主要受数据质量影响。人工放图时建议遵守：

1. 每张图尽量只有一个主体车辆。
2. 车辆主体清楚，轮式结构尽量可见。
3. 删除玩具车、模型图、卡通图、零件图、图标和明显错误类别。
4. 三轮车要特别筛选，避免混入自行车、摩托车、四轮货车或只有局部车身的图片。
5. 三类图片数量尽量接近，避免某一类明显过多。

## 八、跨系统复现说明

该项目可在 macOS、Linux、Windows 上运行。为了提高复现性：

1. 只把人工整理好的图片维护在 `data/raw`。
2. 不手动修改 `data/train` 和 `data/val`。
3. 每次训练前运行 `python3 split_manual_data.py` 重新生成切分。
4. 尽量使用相同 Python、PyTorch、torchvision、Pillow 版本。

不同系统和不同 PyTorch 版本可能导致训练结果略有差异，但数据切分会保持稳定。
